VitaSide

A temporary local AI sidecar that turns Omi notes, Apple Health signals, and journals into explainable lifestyle patterns, what-if simulations, and doctor-ready reports.

See the demo flow Open the repo
Local No cloud sync by default. Personal health data stays on the user's device.
Scoped Read-only access is issued through a manifest with TTL and audit logs.
Useful Patterns, lagged correlations, anomalies, and simulations for better visits.

Doctors see snapshots. VitaSide reconstructs the weeks between them.

Wearables show numbers, journals hold context, and voice notes capture lived reality. VitaSide connects those streams locally so a patient can walk into an appointment with specific observations instead of vague memory.

Voice becomes signal

Omi and journal notes are parsed into dated signals for sleep, stress, food, mood, activity, and symptoms.

Biometrics get context

Sleep, HRV, steps, SpO2, and other Apple Health metrics are compared against the user's own baseline.

Reports stay grounded

Every insight includes confidence, citations, and a clear disclaimer: this is self-observation, not diagnosis.

A small protocol for safe expert intelligence injection.

VitaSide is not another health app. It is a sidecar pattern: a specialist agent issued for a limited purpose, loaded by a personal AI host, and revoked when its job is done.

1

Issue

A doctor or template creates a manifest for a specialist sidecar.

2

Scope

The user grants read-only access to approved local folders and exports.

3

Analyze

The sidecar exposes MCP tools for correlations, anomalies, and simulations.

4

Expire

TTL, audit logs, and local execution keep control with the patient.

Built for a credible hackathon demo, not a mockup.

The repository includes the MCP server, demo data generation, report rendering, a sidecar protocol wrapper, quality gates, and a live script for judges.

MCP-native tools

`analyze_lifestyle_patterns`, `simulate_whatif`, `generate_doctor_report`, `find_correlation`, and source inspection tools.

Local safety model

Manifest scopes, TTL checks, audit trail, no network dependency, and safe fallback to generated demo data.

Pitch-ready output

A 6-8 minute story: issue sidecar, analyze patterns, run what-if, export the report, and explain the extensible protocol.
